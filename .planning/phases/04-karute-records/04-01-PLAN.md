  <name>Task 2: Create Server Actions and query helpers for karute records</name>
  <files>src/actions/karute.ts, src/actions/entries.ts, src/lib/supabase/karute.ts</files>
  <action>
Create `src/actions/karute.ts` as a Server Action file ('use server'):
- `saveKaruteRecord(input: SaveKaruteInput)`: Sequential insert pattern from research.
  1. Insert into karute_records (customer_id, staff_id, transcript, summary, duration) and `.select('id').single()` to get the new ID.
  2. Bulk-insert entries array with karute_record_id set to the new ID. Map fields: category, content, source_quote (from sourceQuote), confidence_score (from confidenceScore), is_manual=false.
  3. On entries insert failure, delete the orphaned karute_record row (cleanup pattern from research).
  4. Call `revalidatePath('/customers/${input.customerId}')`.
  5. Call `redirect('/karute/${recordId}')` OUTSIDE the try/catch block (per research anti-pattern warning).
  Return `{ error: string }` on failure (before the redirect).
  Use `createClient()` from `@/lib/supabase/server` (server Supabase client).

Create `src/actions/entries.ts` as a Server Action file ('use server'):
- `addManualEntry(input: AddEntryInput)`: Insert a single entry row with is_manual=true, confidence_score=null, source_quote=null. Call `revalidatePath('/karute/${input.karuteRecordId}')` after insert. Return `{ error?: string }`.
- `deleteEntry(entryId: string, karuteRecordId: string)`: Delete entry by ID, revalidate path. Return `{ error?: string }`.

Create `src/lib/supabase/karute.ts`:
- `getKaruteRecord(id: string)`: PostgREST nested select fetching karute_records with `customers (id, name)` and `entries (id, category, content, source_quote, confidence_score, is_manual, created_at)` joined. Use `.eq('id', id).single()`. Order entries by `created_at` ascending to avoid pitfall 6 from research.
- Export the inferred type via `QueryData` for type safety.

For staff_id: Since Phase 5 (Staff Profiles) is not yet built, the save action should accept staffId as an optional parameter. If not provided, query for the first profile row as a fallback (or use the auth user ID). Add a TODO comment noting this will be wired to the staff switcher in Phase 5.
