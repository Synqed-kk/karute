         },

     },
   }))

   ```typescript
   jest.mock('openai', () => ({

     toFile: jest.fn().mockResolvedValue('mocked-file'),
   }))
   ```

6. Import `mockTranscriptionResult` from helpers/openai-mocks.

- **"returns transcript for valid audio upload"**: Set `(openai.audio.transcriptions.create as jest.Mock).mockResolvedValue(mockTranscriptionResult)`. Assert status 200, response body has `transcript` string matching mockTranscriptionResult.
- **"returns 400 when no audio file provided"**: POST with empty FormData. Assert status 400, response body has `error` containing 'No audio'.
- **"defaults locale to ja when not specified"**: POST with audio but no locale field. Assert `openai.audio.transcriptions.create` was called with `language: 'ja'`.
- **"handles mp4 audio (iOS Safari)"**: POST with Blob type 'audio/mp4'. Assert `openai.audio.transcriptions.create` was called (verify the mock was invoked — the route handles mp4 extension mapping internally).

IMPORTANT: The transcribe route calls `openai.audio.transcriptions.create` which returns a string directly (response_format: 'text'). The mock should return a plain string, NOT an object.
